# BSD

使用者可以為所欲為,可以自由的使用，修改源代碼，也可以將修改後的代碼作為開源或者專有軟件再發佈。 必須滿足三個條件，才可以*為所欲為,可以自由的使用：

  • 再發佈的產品中包含源代碼，則在源代碼中必須帶有原來代碼中的BSD協議
  • 再發佈的只是二進制類庫/軟件，則需要在類庫/軟件的文檔和版權聲明中包含原來代碼中的BSD協議
  • 不可以用開源代碼的作者/機構名字和原來產品的名字做市場推廣

# Apache(2.0、1.1、1.0)

允許代碼修改，再發佈（作為開源或商業軟件），但需要給代碼的用戶一份Apache Licence，如果你修改了代碼，需要再被修改的文件中說明。若在延伸的代碼中（修改和有源代碼衍生的代碼中）需要帶有原來代碼中的協議，商標，專利聲明和其他原來作者規定需要包含的說明。

# GPL

只要在一個軟件中使用("使用"指類庫引用，修改後的代碼或者衍生代碼)GPL協議的產品，則該軟件產品必須也採用GPL協議，既必須也是開源和免費。

# LGPL

允許商業軟件通過類庫引用(link)方式使用LGPL類庫而不需要開源商業軟件的代碼。這使得採用LGPL協議的開源代碼可以被商業軟件作為類庫引用並發布和銷售。

# MIT

必須在你的發行版裡包含原許可協議的聲明,無論你是以二進制發佈的還是以源代碼發佈的。
