# 快速開始

# 建立資料夾
> mkdir mynote

# 進入資料夾 /mynote
> cd mynote

# 初始化 npm 專案，會出現一些基本設定，可以直接按 enter
> npm init

# 安裝 VuePress
> npm install vuepress # yarn add vuepress

# 使用 vs code 開啟專案
> code .

# 編輯 package.json，在 script 區段加入

"scripts": {
        "docs:dev": "vuepress dev docs",
        "docs:build": "vuepress build docs",
    },

以上範例使用 docs 目錄作為文件入口，請自行修改調整 script 中的參數

# 建立 docs 資料夾，並進入 docs/ 資料夾
mynote > mkdir docs
mynote > cd docs

# 建立 .vuepress 的隱藏資料夾，並進入 .vuepress 資料夾
docs > mkdir .vuepress
docs > cd .vuepress

# 新增 config.js 檔案，建立基本的設定

module.exports = {
   base: "/vuepresstest/", // 設定你的 github 的 repository 名稱 
   title: "vuepress 測試", // 網站 title
   description: "", // 簡介
   themeConfig: { // 主題設定
   // 上方的導航連結
       nav: [
           { text: '首頁', link: '/' },
           { text: 'GitHub', link: 'https://github.com' },
       ],
       // 左側功能表
       sidebar: [
           {
               title: '項目一',   // 必要的
               path: '/',      // 可選的, 应该是一个绝对路径
           },
           {
               title: '項目二',   // 必要的
               path: '/Other',      // 可選的, 应该是一个绝对路径(注意大小寫)
           },

       ]
   }
}

# 撰寫第一份文件

回到 docs 目錄下，新增 Readme.md 檔案(開頭需大寫)，該檔案預設為入口頁面(Index)，內容加入 markdown 語法。

## 我的第一份文件

* item 1
* item 2

預覽結果

> npm run docs:dev

# 部署 github page

  • 建立 Repository，名稱對應 config.js 的 base 設定值。

  • 加入部署設定檔案，在 docs 目錄建立 build 資料夾。

  • 新增 deploy.sh 檔案 (在 linux、Mac 上的腳本，windows 上為 .bat or .ps1)，數入部署腳本。

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
 git push -f https://github.com/<github名稱>/<rpository名稱>.git master:gh-pages

cd -
  • 修改 package.json 加入下方高亮區塊




 




"scripts": {
        "docs:dev": "vuepress dev docs",
        "docs:build": "vuepress build docs",
        "docs:publish": "docs/build/deploy.sh"
    },
  • 開始部署
> npm run docs:publish
  • 瀏覽 pages 頁面，進入 github repository 頁面，點選 settings

  • 往下滾動到 Github Pages 頁面，這就是你的文件Url
